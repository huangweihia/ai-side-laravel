## 一、产品概述

### 1.1 产品定位

**产品名称：** AI 副业情报局  
**Slogan：** 每天 10 分钟，发现 AI 副业机会  
**英文名：** AI Side Hustle Intelligence Bureau

**一句话说明：**
> 一个聚合全网 AI 副业内容（项目/文章/职位），帮助用户发现 AI 赚钱机会的情报平台

### 1.2 目标用户

| 用户类型 | 年龄段 | 特征 | 核心需求 |
|----------|--------|------|----------|
| **上班族** | 25-35 岁 | 有稳定工作，想搞副业但没时间研究 | 高效获取靠谱副业方向，跟着做就能赚钱 |
| **大学生** | 20-24 岁 | 有时间，想赚钱但没经验没预算 | 低成本入门项目，边学边赚 |
| **自由职业者** | 25-40 岁 | 靠技能赚钱，想提升效率/获客 | 提升效率的 AI 工具，获客渠道 |
| **创业者** | 30-45 岁 | 想找 AI 创业方向，有预算 | 行业趋势、竞品分析、商业机会 |
| **技术人员** | 22-40 岁 | 懂技术，想用 AI 做产品 | 技术教程、开源项目、变现案例 |

### 1.3 核心价值主张

**对用户：**
- ✅ **省时间** - 不用到处找，我们帮你聚合全网 AI 内容
- ✅ **避坑** - 每个项目都有变现路径分析和难度评估
- ✅ **可执行** - 不只是资讯，还有完整教程和资源
- ✅ **持续更新** - 每日自动采集，永远有新鲜内容

**对平台：**
- ✅ **内容壁垒** - 自动采集 + 结构化整理，形成内容护城河
- ✅ **付费转化** - 免费内容吸引，VIP 内容变现
- ✅ **网络效应** - 用户越多，数据越精准，推荐越准确

## 三、核心功能详细说明

### 3.1 内容采集系统（P0 - ✅ 已完成）

**功能描述：**
自动从全网采集 AI 相关内容，每次采集 exactly 5 篇不重复文章。

**采集来源详情：**

| 来源 | 账号/频道 | 内容类型 | 数量 | 优先级 |
|------|-----------|----------|------|--------|
| **微信公众号** | 机器之心 | AI 技术/资讯 | 2 篇 | P0 |
| | 量子位 | AI 行业分析 | 1 篇 | P0 |
| | AI 科技大本营 | AI 工具推荐 | 1 篇 | P0 |
| | 菜鸟学 Python | 技术教程 | 1 篇 | P1 |
| **知乎** | AI/副业话题 | 行业分析/经验 | 3 篇 | P1 |
| **小红书** | AI 变现笔记 | 变现案例 | 2 篇 | P1 |
| **36 氪** | AI 频道 | 行业资讯 | 2 篇 | P2 |
| **虎嗅** | AI/商业 | 商业分析 | 1 篇 | P2 |
| **少数派** | 效率工具 | 工具教程 | 2 篇 | P2 |

**采集流程：**
```
1. 打乱所有来源文章顺序（shuffle）
2. 遍历文章列表
3. 检查 source_url 是否已存在（唯一索引）
4. 如果存在，跳过并记录日志
5. 如果不存在，创建文章记录
6. 达到 5 篇后停止
7. 如果遍历完不足 5 篇，继续从下一个来源采集
```

**去重逻辑：**
```php
// 1. 检查 source_url 是否已存在（数据库唯一索引）
$exists = Article::where('source_url', $article['source_url'])->exists();

// 2. 如果已存在，跳过
if ($exists) {
    $this->log("⏭️ 跳过已存在：{$article['title']}");
    continue;
}

// 3. 创建文章
Article::create([...]);
```

## 四、数据库设计详解

### 4.1 核心表结构

#### users（用户表）
```sql
CREATE TABLE `users` (
  `id` bigint UNSIGNED NOT NULL AUTO_INCREMENT,
  `name` varchar(255) NOT NULL,
  `email` varchar(255) NOT NULL,
  `email_verified_at` timestamp NULL DEFAULT NULL,
  `password` varchar(255) NOT NULL,
  `remember_token` varchar(100) DEFAULT NULL,
  `avatar` varchar(255) DEFAULT NULL,
  `role` enum('user','vip','admin') DEFAULT 'user',
  `subscription_ends_at` timestamp NULL DEFAULT NULL,
  `last_login_at` timestamp NULL DEFAULT NULL,
  `last_login_ip` varchar(45) DEFAULT NULL,
  `created_at` timestamp NULL DEFAULT NULL,
  `updated_at` timestamp NULL DEFAULT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `users_email_unique` (`email`),
  KEY `users_role_index` (`role`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**字段说明：**
- `role`: 用户角色（user/vip/admin）
- `subscription_ends_at`: VIP 到期时间（NULL 表示非 VIP）
- `last_login_at/ip`: 最后登录信息（用于安全审计）

**索引优化：**
- 唯一索引：`email`（防止重复注册）
- 普通索引：`role`（快速筛选管理员/VIP）
