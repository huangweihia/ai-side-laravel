# 📋 AI 副业情报局 - 产品需求文档 (PRD)

**版本：** v1.0  
**制定时间：** 2026-03-25  
**产品阶段：** MVP → 成长期  
**目标用户：** 想用 AI 做副业的上班族、大学生、自由职业者

---

## 一、产品定位

### 1.1 一句话说明
**"每天 10 分钟，发现 AI 副业机会"**

### 1.2 核心价值
| 用户痛点 | 我们的解决方案 |
|----------|---------------|
| 不知道 AI 能做什么副业 | 每日精选 AI 副业项目 + 变现路径 |
| 信息太分散，找不到靠谱资源 | 聚合全网 AI 内容（公众号/知乎/GitHub） |
| 想学习但不知道从哪开始 | 结构化知识库 + 智能推荐 |
| 担心被骗，想看看别人怎么做 | 真实案例 + 用户评论 + 社群交流 |

### 1.3 目标用户画像
| 用户类型 | 特征 | 需求 |
|----------|------|------|
| 上班族小王 | 25-35 岁，有稳定工作，想搞副业 | 时间少，需要高效、靠谱的副业方向 |
| 大学生小李 | 20-24 岁，有时间，想赚钱 | 预算少，需要低成本入门项目 |
| 自由职业者小张 | 25-40 岁，靠技能赚钱 | 需要获客渠道、提升效率的工具 |
| 创业者小赵 | 30-45 岁，想找 AI 创业方向 | 需要行业趋势、竞品分析 |

## 三、核心功能详细说明

### 3.1 内容采集系统（P0 - 已完成）

**功能描述：**
自动从全网采集 AI 相关内容，每天至少采集 15 篇不重复文章。

**采集来源：**
| 来源 | 内容类型 | 数量 | 频率 |
|------|----------|------|------|
| 微信公众号 | AI 教程/案例 | 5 篇 | 每日 |
| 知乎 | 行业分析/经验 | 3 篇 | 每日 |
| 小红书 | 变现案例 | 2 篇 | 每日 |
| 36 氪 | 行业资讯 | 2 篇 | 每日 |
| 虎嗅 | 商业分析 | 1 篇 | 每日 |
| 少数派 | 效率工具 | 2 篇 | 每日 |
| GitHub Trending | AI 项目 | 5 个 | 每日 |

**去重逻辑：**
1. 检查 `source_url` 是否已存在（唯一索引）
2. 检查标题是否重复（模糊匹配）
3. 如果重复，自动跳过

**质量要求：**
- 必须有封面图
- 必须有原文链接
- 内容 HTML 格式规范
- 分类准确

**验收标准：**
- [x] 每次采集 exactly 5 篇不重复文章
- [x] 重复文章自动跳过，继续采集新的
- [x] 采集失败有错误日志
- [x] 采集进度实时显示

---

### 3.2 邮件订阅系统（P0 - 已完成）

**功能描述：**
用户可订阅日报/周报/通知，系统定时发送精选内容。

**订阅类型：**
| 类型 | 发送时间 | 内容 | 优先级 |
|------|----------|------|--------|
| 日报 | 每日 8:00 | 当日最新 5 个项目 +5 篇文章 | P0 |
| 周报 | 每周一 10:00 | 本周 Top10 内容汇总 | P1 |
| 通知 | 实时 | 系统通知/活动推送 | P2 |

**退订流程：**
```
用户点击邮件底部退订链接 
  ↓
选择退订类型（可单选/全选）
  ↓
确认退订
  ↓
显示成功页面（可重新订阅）
```

**合规要求：**
- 每封邮件必须有退订链接
- 退订后 24 小时内不能再发送
- 重新订阅需要用户主动操作

**验收标准：**
- [x] 用户可以订阅/退订/重新订阅
- [x] 邮件正确渲染 HTML（不是源代码）
- [x] 支持多选模板发送
- [x] 发送失败有重试机制

### 3.3 VIP 权限系统（P0 - 已完成）

**用户角色：**
| 角色 | 标识 | 权限 |
|------|------|------|
| 普通用户 | `user` | 浏览公开内容、注册登录 |
| VIP 用户 | `vip` | 访问 VIP 内容、专属功能 |
| 管理员 | `admin` | 后台管理、用户管理 |

**权限控制：**
```
前台权限：
- 公开内容：所有用户可访问
- VIP 文章：仅 VIP 可访问全文（非 VIP 看前 100 字）
- VIP 项目：仅 VIP 可查看变现路径

后台权限：
- 仅 admin 角色可访问
- 自动检测角色，非管理员显示 403
```

**VIP 权益设计：**
| 权益 | 免费用户 | VIP 用户 |
|------|----------|----------|
| 文章阅读 | 仅免费文章 | 全部文章 |
| 项目查看 | 基础信息 | 变现路径 + 教程 |
| 知识检索 | 每月 10 次 | 无限次 |
| 邮件订阅 | 仅日报 | 日报 + 周报 + 通知 |
| 收藏功能 | 最多 20 个 | 无限个 |
| 评论功能 | ❌ | ✅ |

**验收标准：**
- [x] 非 VIP 访问 VIP 文章显示模糊遮罩
- [x] 点击"开通 VIP"跳转到 VIP 页面
- [x] VIP 用户直接显示全文
- [x] 后台仅管理员可访问

### 3.4 点赞收藏功能（P1 - 已完成）

**功能描述：**
用户可点赞/收藏喜欢的项目和文章。

**交互流程：**
```
未登录用户点击点赞/收藏 
  ↓
提示"请先登录"
  ↓
跳转登录页

已登录用户点击
  ↓
按钮状态变化（点赞变红/收藏变黄）
  ↓
计数 +1
  ↓
再次点击取消
```

**数据库设计：**
```sql
-- 用户行为表
CREATE TABLE user_actions (
    id BIGINT PRIMARY KEY,
    user_id BIGINT,
    actionable_type VARCHAR,  -- App\Models\Article
    actionable_id BIGINT,
    type VARCHAR,  -- like/favorite
    created_at TIMESTAMP
);

-- 唯一索引：防止重复点赞
UNIQUE INDEX unique_action (user_id, actionable_type, actionable_id, type)
```

**验收标准：**
- [x] 登录用户可点赞/收藏
- [x] 未登录用户提示登录
- [x] 按钮状态实时更新
- [x] 计数准确
- [x] 可取消操作

### 3.5 异步任务系统（P0 - 已完成）

**功能描述：**
后台手动触发采集任务，实时显示进度。

**任务类型：**
| 类型 | 说明 | 频率 |
|------|------|------|
| 文章采集 | 从 6 个来源采集 5 篇文章 | 按需 |
| 项目采集 | 从 GitHub 采集热门项目 | 按需 |
| 知识库采集 | 生成 AI 学习内容 | 按需 |

**交互设计：**
```
点击"采集文章"按钮
  ↓
按钮变灰禁用（loading 状态）
  ↓
等待 5-10 秒（同步执行）
  ↓
显示通知："✅ 成功采集 5 篇文章"
  ↓
按钮恢复可点击
```

**任务记录：**
- 任务名称
- 任务类型
- 开始时间
- 完成时间
- 成功数量
- 失败数量
- 错误信息

**验收标准：**
- [x] 点击后按钮禁用（防止重复点击）
- [x] 采集过程显示进度
- [x] 采集完成显示结果通知
- [x] 任务记录可在后台查看

### 3.6 知识检索系统（P0 - 待开发）

**功能描述：**
用户上传文档，系统建立索引，支持智能检索和 AI 问答。

**核心流程：**
```
用户上传文档（PDF/Word/Markdown）
  ↓
系统解析文本内容
  ↓
分块处理（chunking）
  ↓
向量化存储（阿里云 Embedding）
  ↓
用户搜索问题
  ↓
相似度检索
  ↓
AI 生成综合答案
```

**收费模式：**
| 套餐 | 价格 | 检索次数 | 存储空间 |
|------|------|----------|----------|
| 免费版 | ¥0 | 10 次/月 | 100MB |
| VIP | ¥29/月 | 无限 | 1GB |
| 企业版 | ¥999/月 | 无限+API | 10GB |

**技术实现：**
- 文档解析：Apache Tika / pdfparser
- 向量化：阿里云百炼 Embedding
- 存储：Milvus / Pinecone
- 检索：相似度搜索 + 关键词搜索

**开发优先级：** P0（核心收费功能）

## 八、风险与应对

### 8.1 合规风险
**风险：** 支付牌照、内容审核

**应对：**
- 咨询律师，确保合规
- 内容审核机制
- 用户举报通道

### 8.2 竞争风险
**风险：** 类似平台较多（知识星球、小报童等）

**应对：**
- 差异化定位（知识检索）
- 独家高质量内容
- 社群运营

### 8.3 技术风险
**风险：** API 依赖、稳定性

**应对：**
- 多 API 备份
- 本地缓存
- 降级策略

### 8.4 运营风险
**风险：** 用户留存、内容质量

**应对：**
- 持续输出高质量内容
- 用户反馈机制
- 定期活动运营
