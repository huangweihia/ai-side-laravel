## 1. 概述

- **运行方式：** Docker Compose（PHP、Nginx、MySQL、Redis、队列与调度容器等，以项目根目录 `docker-compose.yml` 为准）。
- **适用读者：** 需要在 Windows 本地或 Linux 服务器上部署本 Laravel 项目的维护者。
- **命令约定：** 下文默认使用 Docker Compose V2 语法 `docker compose`。若本机仅有旧版，请将 `docker compose` 替换为 `docker-compose`。

---

## 2. 环境准备

### 2.1 Docker Desktop（Windows）

1. 安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)，推荐使用 WSL2 后端。
2. 启动 Docker Desktop，待引擎就绪（状态为 Running）。
3. 验证：

```powershell
docker --version
docker compose version
```

### 2.2 Git（可选）

用于克隆仓库：[https://git-scm.com/download/win](https://git-scm.com/download/win)

```powershell
git --version
```

## 4. 本地 Docker 部署

### 4.1 一键部署（推荐）

在项目根目录执行（仓库若提供 `deploy-windows.ps1`）：

```powershell
cd D:\lewan\ai-side-laravel
.\deploy-windows.ps1
```

脚本一般会：检查 Docker、复制 `.env`、启动容器、提示执行迁移等。具体以脚本内容为准。

### 4.2 手动部署

```powershell
cd D:\lewan\ai-side-laravel

# 1. 环境文件
Copy-Item .env.example .env

# 2. 启动容器
docker compose up -d

# 3. 等待 MySQL 就绪（首次建议等待约 30 秒）
Start-Sleep -Seconds 30

# 4. 应用初始化（在 php 容器内执行）
docker compose exec php php artisan key:generate
docker compose exec php php artisan migrate
docker compose exec php php artisan make:filament-user
```

创建 Filament 管理员时按提示填写姓名、邮箱、密码（密码建议不少于 8 位）。

若 `docker compose exec php` 报错，先确认服务名是否为 `php`：

```powershell
docker compose ps
```

## 5. 环境变量要点

在 `.env` 中至少确认：

| 项 | 说明 |
|----|------|
| `APP_URL` | 与浏览器访问地址一致；生产环境须为 HTTPS 根地址。 |
| `APP_KEY` | 执行 `php artisan key:generate` 后自动生成。 |
| `DB_*` | 与 `docker-compose.yml` 中 MySQL 配置一致。 |
| `REDIS_*` | 若使用队列/缓存，与 Compose 中 Redis 一致。 |
| `QUEUE_CONNECTION` | 生产常见为 `redis`；本地可按项目默认。 |

复制 `.env` 后**切勿**将含真实密码的 `.env` 提交到 Git。

## 6. 初始化与 Artisan 命令

所有 `artisan` 命令建议在 **PHP 容器**内执行：

```powershell
docker compose exec php php artisan migrate
docker compose exec php php artisan config:clear
docker compose exec php php artisan cache:clear
docker compose exec php php artisan view:clear
```

更新代码或修改 `.env`、`config/` 后建议清理配置缓存并视情况重启 `php` 容器：

```powershell
docker compose restart php
```

## 8. 目录权限与存储

### 8.1 Laravel 常规

在容器内保证 `storage`、`bootstrap/cache` 可写（示例）：

```powershell
docker compose exec php chown -R www-data:www-data /var/www/html/storage /var/www/html/bootstrap/cache
```

路径以容器内项目路径为准（常见为 `/var/www/html`）。

### 8.2 头像上传

头像保存在 `public/avatars/`，需存在且可写：

```powershell
docker compose exec php mkdir -p public/avatars
docker compose exec php chmod 777 public/avatars
```

前台在登录后访问「个人中心」上传；路由一般为 `POST /profile/upload-avatar`。布局中需包含 CSRF：`<meta name="csrf-token" content="{{ csrf_token() }}">`。

**验证规则（参考）：** `image`，`jpeg/png/jpg/gif`，最大约 2MB。若上传失败，先清缓存并检查目录权限与浏览器控制台 Network 状态码（419 多为 CSRF，413 为文件过大）。

## 9. 可选模块：知识检索（知识库）

若项目已包含知识库相关迁移与路由（`/knowledge` 等），在完成基础迁移后：

1. 执行：`docker compose exec php php artisan migrate --force`
2. 按业务需要插入 `knowledge_bases`、`knowledge_documents` 等测试数据（可用 `php artisan tinker`）。
3. **MCP / 外网搜索**：若需对接外部检索，在 `.env` 中配置相应 URL 与密钥（**勿将真实密钥写入文档或提交仓库**）。示例占位：

```env
MCP_WEBSEARCH_URL=
MCP_WEBSEARCH_KEY=
```

4. 生产环境建议：`QUEUE_CONNECTION=redis`，并保证 `storage` 可写；定时任务使用系统 crontab 调用 `php artisan schedule:run` 或与当前部署方式一致。

## 11. 生产环境要点

1. **HTTPS**：全站 TLS，Cookie `secure`、HSTS 等与框架及反向代理配置一致。  
2. **`.env`**：`APP_ENV=production`，`APP_DEBUG=false`。  
3. **调度与队列**：按 `docker-compose.yml` 或进程管理器保持 `schedule:work` / `queue:work` 长期运行（若项目使用队列）。  
4. **备份**：定期备份数据库与 `storage/app` 等用户上传内容。  
5. **防火墙**：仅开放 80/443（及 SSH），数据库、Redis 不暴露公网。

## 12. 上线验证清单

部署或发版后建议逐项确认：

**基础**

- [ ] `docker compose ps`（或等价命令）中相关容器均为 Up（或服务器上服务正常）。
- [ ] `php artisan migrate` 无报错（新环境注意备份后再迁移）。
- [ ] 前台首页可打开，后台 `/admin` 可登录。

**配置与缓存**

- [ ] 已执行 `config:clear`、`cache:clear`、`view:clear`（发版脚本可自动执行）。
- [ ] 修改 `.env` 后已重启 PHP 或重载 PHP-FPM。

**功能抽样**

- [ ] 用户注册/登录。
- [ ] 文章或项目详情：点赞、收藏（若已启用）。
- [ ] 个人中心：头像上传、收藏/评论/浏览列表展示（视功能开启情况）。
- [ ] 邮件相关：若使用 SMTP，后台发一封测试邮件（见 `docs/04-邮件系统/`）。

**微信付费（若已开启 `WECHAT_PAY_ENABLED`）**

- [ ] 测试一笔沙箱或小额订单，回调可达且订单状态正确。

**运营展示（迁移含 `announcements`、`ad_slots` 时）**

- [ ] 后台「运营与数据」可编辑顶部公告与广告位；前台出现滚动公告条（至少一条启用中的公告）与**右侧边栏**推广卡片（需开启 `is_enabled`，并执行 `2026_03_29_150000_add_sidebar_fields_to_ad_slots_table` 迁移；本地上传图需 `php artisan storage:link`）。

## 13. 故障排查

### 13.1 容器无法启动

```powershell
docker compose down
docker compose up -d
docker compose logs -f
```

确认 Docker Desktop（本机）已运行；冲突时检查端口占用（`netstat -ano | findstr :8081`）。

### 13.2 数据库 Connection refused

等待 MySQL 容器完全启动后重试迁移；必要时 `docker compose restart mysql`，等待数秒再执行 `migrate`。

### 13.3 页面 419

多为 CSRF 失效：刷新页面、确认布局中有 `csrf-token` meta；检查会话与域名是否一致。

### 13.4 页面 500

查看日志：`docker compose logs php`、`storage/logs/laravel.log`。

### 13.5 头像/上传失败

确认 `public/avatars` 权限、文件大小与格式；清理缓存后重试。

### 13.6 配置未生效

执行 `php artisan config:clear`，修改 `config/` 或 `.env` 后重启 `php` 容器。
