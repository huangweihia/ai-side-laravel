## 1. 背景与目标

1. **评论不应全站下线**：仅针对「VIP 文章 + 当前用户无权读全文」的场景隐藏评论区；非 VIP 文章、或已具备全文权限的用户仍正常使用评论。
2. **接口与页面一致**：服务端在发表评论时校验全文权限，避免绕过前端隐藏逻辑刷评论。
3. **作者与被回复者感知互动**：通过既有 `system_notifications` 体系写入通知，与点赞、收藏通知风格一致。

---

## 2. 业务规则

### 2.1 是否展示评论区（前端）

定义布尔变量 **`$showComments`**（控制器传入视图）：

```text
showComments = (!article.is_vip) OR canViewFullArticle
```

（与代码中一致：`$showComments = ! $article->is_vip || $canViewFullArticle`。）

含义：

- **非 VIP 文章**：始终展示评论区（含未登录用户可见列表；发表需登录，与项目详情等页一致）。
- **VIP 文章**：仅当 `Article::userCanViewFullContent($user)` 为真时展示（含 VIP 会员、积分解锁、文章作者、管理员等，与正文权限一致）。

### 2.2 发表评论（接口）

- 路由：`POST /articles/{id}/comments`（命名路由 `articles.comments.store`）。
- 除登录外，须满足 **`userCanViewFullContent`**，否则返回 **403**，提示开通 VIP 或解锁后可参与评论。

### 2.3 系统通知

| 通知类型 `type` | 接收人 | 触发条件 | 去重/例外 |
|-----------------|--------|----------|-----------|
| `article_commented` | 文章作者 | 任意用户发表评论或回复，且评论者不是作者 | 无作者 `author_id` 不发 |
| `comment_replied` | 被回复评论的作者 | 存在 `parent_id`（楼中楼），且被回复者不是评论者本人 | 若被回复者即文章作者，则不再发（避免与上一条重复） |

通知 `meta` 建议字段（实现已包含）：`article_id`、`actor_id`、`comment_id`、`parent_id`（视类型而定）。  
前台通知列表中若存在 `meta.article_id`，沿用现有「查看文章 →」链接（`route('articles.show', …)`）。

## 3. 技术实现索引

### 3.1 路由

- 文件：`routes/web.php`  
- 在 `auth` 分组内：`ArticleController@storeComment` → `articles.comments.store`。

### 3.2 控制器

- 文件：`app/Http/Controllers/ArticleController.php`
- **`show`**：计算 `$canViewFullArticle`、`$showComments`；在 `$showComments` 为真时加载评论列表、精选评论、`likedCommentIds` 等（逻辑对齐项目详情评论加载方式）。
- **`storeComment`**：校验权限与父子评论归属；创建评论后调用 `SystemNotificationService` 的 `notifyArticleCommented`、`notifyCommentReplied`（有父评论时）。

### 3.3 视图与脚本

- 文件：`resources/views/articles/show.blade.php`
- 整块评论 UI 包裹在 `@if($showComments)` 内。
- 前端提交使用命名路由 `articles.comments.store`；脚本内函数名与项目详情区分（如 `submitArticleComment`、`startArticleReply` 等），避免冲突。

### 3.4 服务层

- 文件：`app/Services/SystemNotificationService.php`
- 新增方法：`notifyArticleCommented`、`notifyCommentReplied`（依赖 `Comment` 模型与 `Str::limit` 摘要）。

### 3.5 数据与类型说明

- 表：`system_notifications`（迁移注释中已列举 `article_commented` / `comment_replied` 等类型，与实现一致）。
- 评论仍为多态：`comments.commentable_type` / `commentable_id` 指向 `Article`。

## 4. 自测建议（手工）

1. **非 VIP 文章**：未登录可见评论列表；登录可发表、回复、点赞评论。
2. **VIP 文章 + 无全文权限**：不显示评论区；直接 POST 评论接口应 403。
3. **VIP 文章 + 有全文权限**：评论区正常；发表评论后作者账号在「系统通知」可见 `article_commented`。
4. **回复非作者的评论**：被回复用户收到 `comment_replied`；若被回复者为文章作者，仅收到 `article_commented`。
