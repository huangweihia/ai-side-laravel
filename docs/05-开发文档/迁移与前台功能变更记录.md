## 1. 部署必做：执行迁移

若线上/本地数据库**未**执行最新迁移，会出现：

- `SQLSTATE[42S02]: Table 'xxx.system_notifications' doesn't exist`
- 访问「系统通知」等依赖该表的功能时报错

**处理方式**（在项目根目录、使用正确 `.env` 数据库配置）：

```bash
php artisan migrate
```

生产环境可加 `--force`：

```bash
php artisan migrate --force
```

**说明**：代码已增加防护——当表不存在时，前台「系统通知」列表会返回 **503** 与友好说明页（`notifications/setup-required`），避免直接抛出 Flare `QueryException`；**根本解决仍是执行迁移**。

线上/新环境执行 `php artisan migrate` 后，以下 **模板键（key）** 应全部存在，与后台「基础模版」列表一致；**无需依赖 Seeder** 即可具备定时任务 `emails:send-scheduled` 所需的 `daily_digest`。

| `notification` | 【通知】系统通知 | 同上 |
| `profile_message_urgent` | 主页留言 · VIP 紧急通知 | `2026_03_28_100001_add_profile_message_urgent_email_template.php` |
| `vip_expiry_reminder` | VIP 到期提醒 | `2026_03_28_120000_add_vip_expiry_reminder_email_template.php` |

## 8. 邮件订阅：后台列表与前台保存方式

| 层级 | 说明 |
|------|------|
| **数据表** | `email_subscriptions` |
| **模型** | `App\Models\EmailSubscription` |
| **后台** | `App\Filament\Resources\SubscriptionResource`（导航名「邮件订阅」），与前台偏好、注册、邮件任务使用同一套数据 |
| **勿混淆** | `App\Models\Subscription` 面向付费会员套餐（`subscriptions` 表），与邮件订阅不是同一张表；若需后台管理付费订阅应另建资源 |

前台「订阅偏好」页（`SubscriptionController@updatePreferences`）：

- 默认使用 **AJAX**：`fetch` + `Content-Type: application/json` + `Accept: application/json`，无整页刷新。
- 关闭 JavaScript 时，表单仍可按传统 POST 提交（`x-www-form-urlencoded`），走重定向与 `session('success')`。
- 视图：`resources/views/subscriptions/preferences.blade.php`。

## 9. 后台 Filament 菜单结构（固定分组顺序）

在 `AdminPanelProvider` 中注册分组顺序；各 Resource/Page 通过 `navigationGroup` + `navigationSort` 归入组内排序。

| 分组 | 菜单项（排序从小到大） |
|------|-------------------------|
| **内容管理** | 投稿审核 → 文章 → 项目 → 分类 → 评论 → 职位 → 知识库 |
| **用户与互动** | 用户管理 → 积分管理 → 行为日志 → 收藏管理 → 系统通知 |
| **订阅与会员** | 邮件订阅（`EmailSubscription`）→ 付费会员（`Subscription`，路由 slug `paid-subscriptions`） |
| **邮件系统** | 邮件管理 → 邮件模板 → 邮件配置 → SMTP 配置 → 邮件日志 |
| **运营与数据** | 数据报表 |
| **系统** | 任务管理 → 系统设置 |

**付费与邮件区分**

- `subscriptions` 表 + `App\Models\Subscription`：付费 VIP 订单（待支付 / 已生效 / 过期 / 取消），后台 **付费会员**。
- `email_subscriptions` 表 + `App\Models\EmailSubscription`：邮件日报/周报/通知偏好，后台 **邮件订阅**。
