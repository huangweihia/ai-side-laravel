## 1. 部署必做：执行迁移

若线上/本地数据库**未**执行最新迁移，会出现：

- `SQLSTATE[42S02]: Table 'xxx.system_notifications' doesn't exist`
- 访问「系统通知」等依赖该表的功能时报错

**处理方式**（在项目根目录、使用正确 `.env` 数据库配置）：

```bash
php artisan migrate
```

生产环境可加 `--force`：

```bash
php artisan migrate --force
```

**说明**：代码已增加防护——当表不存在时，前台「系统通知」列表会返回 **503** 与友好说明页（`notifications/setup-required`），避免直接抛出 Flare `QueryException`；**根本解决仍是执行迁移**。
